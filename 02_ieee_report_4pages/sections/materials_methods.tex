\section{Materials and Methods}

\subsection{Dataset and Preprocessing}

NephroVision is trained and evaluated on the KiTS23 dataset, which provides contrast-enhanced abdominal CT volumes in NIfTI format with expert annotations for background, kidney, and tumor/cyst. The merged tumor/cyst class follows the challenge convention. Final evaluation uses a project-defined held-out subset of 64 independent cases, never used during training or validation.

Preprocessing comprises HU clipping to $[-200, 300]$ and min--max normalization to $[0, 1]$: $x_{\text{norm}} = (x + 200)/500$. This standardizes input distributions across scanners. Identical preprocessing is applied during training and inference.

\subsection{Model and Training}

The core model is a 3D U-Net with encoder-decoder architecture and skip connections. Three-dimensional convolutions capture inter-slice spatial context essential for kidney morphology. The model outputs three classes: background, kidney, and tumor/cyst.

Training uses a subset of KiTS23 cases with a held-out validation set for checkpoint selection. The best-performing checkpoint on validation Dice is retained for final evaluation. Detailed training hyperparameters are documented in the experiment log.

\subsection{Inference and Post-Processing}

Full volumes are processed via sliding-window inference with $64 \times 192 \times 192$ voxel patches and $50\%$ overlap. Predictions in overlap regions are averaged. Eight-flip test-time augmentation (all combinations of flips along $x$, $y$, $z$ axes) reduces boundary noise; predictions from all augmented versions are averaged voxel-wise.

Post-processing applies connected-component filtering: kidney components $< 5000$ voxels and tumor components $< 100$ voxels are removed. The conservative tumor threshold preserves small true lesions while removing noise, prioritizing detection sensitivity for decision-support use.

\subsection{Output}

The pipeline produces: (1) a 3D segmentation mask in NIfTI format; (2) kidney and tumor/cyst volumetric statistics (ml); (3) an interactive browser-based 3D mesh viewer with rotate, zoom, pan, and class toggling. Fig.~\ref{fig:pipeline} illustrates the end-to-end pipeline.
